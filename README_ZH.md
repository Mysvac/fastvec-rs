# FastVec: SBO Vector

一个为小数据优化的动态数组库。

小缓冲优化（Small-Buffer-Optimization）通过在栈上内联一个小型缓冲区，
在小数据量时避免堆内存分配，从而提升运行效率。

本库提供了三种不同的动态数组，以适应不同场景的需求。

基准测试结果请参考 [benchmarck](./benchmark/README.md) 。

---

## ArrayVec

**完全内联的动态数组**，最大容量在编译期确定。

```rust
let mut vec: ArrayVec<i32, 10> = ArrayVec::new();

vec.push(1);
vec.push(2);

assert_eq!(vec, [1, 2]);
assert_eq!(vec.len(), 2);
assert_eq!(vec.capacity(), 10); // 固定容量
```

类似于 `[T; N]`，但内部使用 `len` 字段记录有效元素数量，允许末尾元素未初始化。
其 API 基本复刻了标准库 `Vec`，任何导致容量溢出的操作都会触发 Panic。

你也可以使用 `Box<ArrayVec>` 的方式将其存储在堆区，实现最大容量固定的 `Vec`。

### 特点

- ✅ **零额外开销**：数据访问无需分支判断，与原生数组效率相同
- ✅ **无额外堆分配**：数据完全内联在对象中
- ❌ **容量固定**：超出 `N` 时无法扩容

此容器和 `arrayvec` 库提供的 `ArrayVec` 类似。

### Feature

如果你不需要此容器，可以关闭本库 `arrayvec` 特性（默认开启），此时相关代码不会编译。

## SmallVec

**传统的小缓冲优化数组**，通过标志位判断数据在堆区还是内联区。

```rust
let mut vec: SmallVec<i32, 2> = SmallVec::new();

vec.push(1);  // 栈上存储
vec.push(2);  // 栈上存储
vec.push(3);  // 超出容量，自动迁移到堆上

assert_eq!(vec, [1, 2, 3]);
```

### 特点

- ✅ **空间高效**：`SmallVec<u64, 2>` 与 `Vec<u64>` 大小相同
- ✅ **自动扩容**：超出内联容量后无缝切换到堆分配
- ✅ **冷路径优化**：使用 `#[cold]` 标记堆路径，提升小数据访问效率
- ⚠️ **每次访问需分支判断**：读取数据时需检查存储位置
- ⚠️ **堆访问性能略低**：大数据场景下，每次访问都是冷路径

此容器和 `smallvec` 的 `SmallVec` 类似。
二者的空间占用基本相同，但本库的 `SmallVec` 通过 `#[cold]` 标记堆路径。
这使得本库的 `SmallVec` 在小数据场景下访问速度更快，但在大数据场景下更慢。

具体差距请参考[基准测试](./benchmark/README.md)的结果。

### Feature

如果你不需要此容器，可以关闭本库 `smallvec` 特性（默认开启），此时相关代码不会编译。

## FastVec

**速度优化版小缓冲数组**，通过指针缓存避免每次访问的分支判断。

```rust
let mut vec: FastVec<i32, 2> = FastVec::new();
let data = vec.data();  // 获取可操作句柄

data.push(1);
data.push(2);
data.push(3);  // 自动迁移到堆

assert_eq!(data.as_slice(), &[1, 2, 3]);
```

`FastVec` 内部维护一个始终指向当前数据位置的缓存指针，所有操作直接通过该指针进行，无需检查数据在内联区还是堆区。
这保证了 `FastVec` 的数据访问速度能与标准库 `Vec` 持平，无论数据的多少。

但注意，增减元素时依然有额外的判断开销（实际开销很低，可参考[基准测试](./benchmark/README.md)的结果）。

### 特点

- ✅ **访问速度恒定**：无论数据在堆还是栈，访问性能与 `Vec` 一致
- ✅ **大数据友好**：切换到堆后性能不劣化
- ✅ **零额外分支**：缓存指针消除了每次访问的条件判断
- ⚠️ **自引用结构**：内部指针可能指向自身栈帧，移动后需刷新
- ⚠️ **非 `Sync`**：因内部使用 `Cell` 维护指针，无法安全跨线程共享

### 工作原理

`FastVec` 采用双类型设计解决自引用问题：

- **`FastVec`**：可移动的包装类型，负责指针刷新
- **`FastVecData`**：实际数据操作类型，通过 `FastVec::data()` 获取

```rust
let mut vec: FastVec<i32, 4> = [1, 2, 3].into();
let data = vec.data();  // 获取数据句柄，指针自动刷新

// 所有操作通过 data 进行，无额外开销
data.push(4);
data.retain(|x| *x % 2 == 0);
```

### Feature

如果你不需要此容器，可以关闭本库 `fastvec` 特性（默认开启），此时相关代码不会编译。

## 选择指南

|     特性     |   ArrayVec   |   SmallVec   |   FastVec   |
|--------------|--------------|-------------|--------------|
| **元素容量** | 固定，不可扩容 | 可扩容到堆   | 可扩容到堆    |
| **空间效率** | 高           | 高           | 中（额外指针  |
| **访问开销** | 零           | 分支判断      | 零（缓存指针）|
| **堆性能**   | 不适用       | 较低（冷路径）| 与 Vec 相同   |
| **Sync**    | ✅           | ✅          | ❌          |
| **适用场景** | 数据量有上限  | 空间敏感      | 性能敏感     |

- **最大数据量确定，追求性能** → 使用 `ArrayVec`
- **数据量通常较小，空间敏感或用于数据存储** → 使用 `SmallVec`
- **数据量通常较小，追求性能，临时处理数据** → 使用 `FastVec`
